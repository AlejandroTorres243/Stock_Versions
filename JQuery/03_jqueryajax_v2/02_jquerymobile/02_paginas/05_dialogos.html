Dialogos

Creando dialogos

todas las paginas pueden ser presentadas como un dialogo modal con data-rel="dialog" automaticamente se formatea para darle el aspecto de dialogo

Abrir dialogo

Transiciones

Por defecto se abre con 'pop', se puede especificar, en todos. Se recomienda usar pop slidedown o flip.

data-transition="pop" data-transition="slidedown" data-transition="flip"

Cerrar los dialogos

HAbitualmente cuando cualquier link en un dialogo es pulsado, el framework. tambien se puede cerrar añadiendo un boton con data-rel="back".

Si se crean paginas con javascript, a parte del data-rel="back" que se podria añadir automaticamente, tambien se puede cerrar con el metodo close(), por ejemplo: $('.ui-dialog').close()

Estableciendo el valor del "boton volver"

Al igula que en cualquier pagina, se puede poner este valor mediante una opcion o un atributo. La opcion puede ser establecida para todos los dialogos dentro del evento mobileinit, y establecidendo $.mobile.dialog.prototype.options.closeBtnText = "mitexto", o tambien, se puede utilizar el atributo data-close-btn-text

Historial

Los dialogos no se incluyen en el historial

Margenes

Por defecto todos los dialogos tienen un max-width de 500px y 15px de padding a cada lado y un 10% de margin top, pero se puede modificar esto de la siguiente forma

.ui-dialog .ui-header, 
.ui-dialog .ui-content, 
.ui-dialog .ui-footer { 
	max-width: 500px; 
	margin: 10% auto 15px auto; 
}
